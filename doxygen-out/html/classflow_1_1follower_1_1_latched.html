Detailed Description

template<typename DispatchT, typename LockPolicyT = NoLock, typename ContainerT = DefaultContainer<DispatchT>, typename QueueMonitorT = DefaultDispatchQueueMonitor>
class flow::follower::Latched< DispatchT, LockPolicyT, ContainerT, QueueMonitorT >

Captures one element before the capture range lower bound, minus a minimum period. 

   All older elements are removed. If no newer elements are present on the next capture attempt,
   then the last captured element is returned. If a newer element is present on a subsequent capture attempt,
   meeting the aforementioned qualifications, this elements is captured and replaces "latched" element state.
Template Parameters
DispatchTdata dispatch type
LockPolicyTa BasicLockable (https://en.cppreference.com/w/cpp/named_req/BasicLockable) object or NoLock or PollingLock
ContainerTunderlying DispatchT container type
QueueMonitorTobject used to monitor queue state on each insertion; used to precondition capture
Note
Latched won't behave non-deterministically if actual input period (difference between successive dispatch stamps) is greater than min_period. However, newer data values will not be captured if they are within a min_period offset of range.lower_stamp

@warn Latched may never enter a READY state if data never becomes available. Calling application may need to implement a synchronization timeout behavior
